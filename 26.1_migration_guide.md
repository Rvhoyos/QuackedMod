# Context: Minecraft 26.1 (Tiny Takeover) Migration Guide
**Target Loaders:** Fabric & NeoForge (Multiloader Architecture)
**Key Dependency:** GeckoLib (26.1+ compatible version required)

## 1. Environment & Toolchain
- **Java Runtime:** Upgrade to **Java 25** (Mandatory for 26.1).
- **Gradle:** Recommended upgrade to **Gradle 9.x**.
- **De-obfuscation:** Mojang has removed obfuscation. 
    - **Action:** Remove `Parchment` or `Yarn` mapping dependencies. 
    - **Action:** Use official Mojang parameter names natively.

## 2. Breaking API Changes (Common Code)

### A. ItemStack vs. ItemStackTemplate
The `ItemStack` class is now restricted for "live" world-bound items only.
- **Change:** Static item definitions or "blueprints" must use `ItemStackTemplate`.
- **Logic:** Replace `new ItemStack(Items.DIRT)` with the template equivalent.
- **Method:** Use `template.createStack(int count)` to instantiate a stack once a Level/World is available.

### B. GUI & Rendering (The Extractor Refactor)
Rendering is now decoupled to support the **Vulkan API**.
- **Refactor:** `GuiGraphics` calls in `Screen` are largely replaced by `GuiGraphicsExtractor`.
- **Key Method Renames:**
    - `Screen#render` -> `Screen#extractRenderState`
    - `Screen#renderBackground` -> `Screen#extractBackground`
- **Vertex Buffers:** Any direct GL calls or custom BufferBuilders must be Vulkan-compliant.

### C. Data-Driven Systems
- **Villager Trades:** Hardcoded trade registries are deprecated. Trades are now 100% defined via **JSON Datapacks**.
- **Fluids:** `FluidRenderHandler` (Fabric) and NeoForge equivalents are replaced by the `FluidModel` registration system.

## 3. Loader-Specific Notes
- **NeoForge 26.1:** Requires the new `NeoForm` toolchain which handles the de-obfuscated JAR.
- **Fabric:** Ensure `fabric-loom` is updated to the latest version to support the native Mojang mappings.

## 4. External Dependencies
- **GeckoLib:** Must use the 26.1-specific release. The internal rendering engine was rewritten for the Vulkan transition. API signatures remain similar, but bytecode is incompatible with 1.21.x.
